# About

I'm a technical writer with 7+ years of experience creating and managing product documentation, currently focused on API and developer documentation.

I discovered technical writing early in my career when I was asked to join a documentation team at an EdTech company — and it changed my trajectory completely. I found that I genuinely enjoy figuring out how things work and then sharing what I've learned with others. I earned a reputation for taking on the most technically complex projects, including things like mapping out exactly how software permissions interacted across an entire product suite.

When I decided to focus on developer documentation, I committed to learning it the way I learn best: by actually doing it. I completed coursework in REST API documentation, OpenAPI/Swagger specifications, and Git-based workflows, built API documentation portfolio pieces from scratch using Postman and ReadMe, and got hands-on with the tools and workflows currently popular in the field — including Git, Docusaurus, and docs-as-code pipelines.

I'm currently looking for a remote technical writer role focused on API and developer documentation, where I can create content that makes users at any level feel like it was written for them.
